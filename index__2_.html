Boost Converter Simulation

Schematic
// LTspice schematic — 5V in, 100µH, 100µF, 20Ω load
Specs
Input5V DC
Output10V DC
Duty Cycle50%
SwitchNMOS (PWM)
Inductor100µH
Capacitor100µF
Load20Ω
The Problem

First run showed a 13.5V startup spike — 35% above target. The inductor was dumping stored energy into the capacitor all at once, with no ramp-up time to control it.

Output — Before vs After
Before — 13V spike, unstable
After — 11V spike, settles to ~10V
The Fix

Enabled LTspice's .startup command to ramp the 5V source from zero — dropped the spike from 13.5V to ~11V. Then upsized inductor and capacitor from 10µ to 100µ to absorb the remaining energy surge.

03 / 04 — LTspice

Half-Wave Rectifier — LED Load

Schematic
// SINE(0 5 60) source, 1N914 diode, 100µF cap, 3× LED loads
The Problem

Without a filter capacitor, the output had large gaps where voltage dropped to zero between positive humps. Three LED loads would have flickered at 60Hz — unusable for any sensitive application.

Output Waveform
// V(n002) — filtered DC output with 100µF capacitor, ~4.1V peak
The Fix

Added a 100µF capacitor in parallel with the loads. It charges during each positive cycle and holds charge through the gaps — turning a choppy pulsed signal into smooth near-constant DC.

Key Takeaway

A diode converts AC to DC. A capacitor is what makes that DC actually usable. Next step: full-wave rectifier to halve the gaps and reduce ripple further.

04 / 04 — LTspice

AC-to-5V Power Supply — Arduino Uno Model

Schematic
// Bridge rectifier → 4700µF bulk cap → BS170 MOSFET buck → 25Ω Arduino Uno model
Specs
Input20V AC, 60Hz
Output5V DC
Duty Cycle65%
Rectifier4x 1N4007
Bulk Cap4700µF
SwitchBS170 MOSFET
Inductor100µH
Filter Cap100µF
Load25Ω (Arduino model)
The Problem

When the bridge rectifier was connected directly to the buck converter, the output followed the AC humps instead of staying flat. The buck converter was seeing the rectified signal collapse to zero every cycle — nothing stable to regulate from.

Output Waveform
// V(n004) — output ramps up and locks to a flat 5V DC
The Fix

Added a 4700µF bulk capacitor between the bridge rectifier output and the MOSFET drain. It charges to the peak rectified voltage and holds that charge during the gaps between AC humps — giving the buck converter a stable DC rail to regulate from.

Key Takeaway

The duty cycle ended up at 65% rather than the theoretical 35% due to real component losses from the bridge diodes and MOSFET on-resistance — a good reminder that real components never behave as ideally as the math suggests. A bridge rectifier converts AC to DC, but without a bulk capacitor the buck converter has nothing stable to work with.
